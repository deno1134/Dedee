# 🚀 Render.com Deployment Guide

## 📋 Prerequisites

- ✅ GitHub repository with your code
- ✅ Render.com account
- ✅ PostgreSQL database ready
- ✅ Environment variables configured

## 🔧 Step-by-Step Deployment

### 1. **PostgreSQL Database Setup**

#### Create Database on Render:
```bash
# Go to Render Dashboard
# 1. Click "New" → "PostgreSQL"
# 2. Name: denicord-db
# 3. Database: denicord  
# 4. User: denicord_user
# 5. Region: Frankfurt (EU)
# 6. Plan: Free
# 7. Click "Create Database"
```

#### Deploy Schema:
```bash
# Wait for database to be ready
# Copy External Database URL from Render dashboard
export DATABASE_URL="postgresql://denicord_user:password@host:5432/denicord"

# Deploy schema
psql $DATABASE_URL -f server/database/schema.sql
```

### 2. **Backend Server Deployment**

#### Create Web Service:
```bash
# 1. Go to Render Dashboard
# 2. Click "New" → "Web Service"
# 3. Connect your GitHub repository
# 4. Select branch: main
# 5. Root Directory: server
# 6. Environment: Node
# 7. Build Command: npm install
# 8. Start Command: npm start
```

#### Environment Variables:
```env
NODE_ENV=production
DATABASE_URL=[Link to PostgreSQL database]
JWT_SECRET=[Auto-generated by Render]
JWT_EXPIRES_IN=7d
CORS_ORIGIN=https://denicord-frontend.onrender.com
MAX_FILE_SIZE=50MB
UPLOAD_DIR=/tmp/uploads
LOG_LEVEL=info
```

#### Advanced Settings:
```bash
# Health Check Path: /api/health
# Auto-Deploy: Yes
# Region: Frankfurt
# Plan: Free
```

### 3. **Frontend/Desktop Web App Deployment**

#### Create Static Site:
```bash
# 1. Go to Render Dashboard
# 2. Click "New" → "Static Site"
# 3. Connect your GitHub repository
# 4. Branch: main
# 5. Root Directory: desktop
# 6. Build Command: npm install && npm run build
# 7. Publish Directory: dist
```

#### Environment Variables:
```env
VITE_API_URL=https://denicord-backend.onrender.com/api/
VITE_WS_URL=wss://denicord-backend.onrender.com
```

### 10. **Performance Optimization**

#### Database Optimization:
```sql
-- Add indexes for better performance
CREATE INDEX CONCURRENTLY idx_messages_channel_created 
ON messages(channel_id, created_at DESC);

CREATE INDEX CONCURRENTLY idx_users_status 
ON users(status) WHERE status != 'offline';
```

#### CDN Configuration:
```javascript
// Add to server.js
const express = require('express');
const compression = require('compression');
const rateLimit = require('express-rate-limit');

app.use(compression());
app.use(rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 100 // limit each IP to 100 requests per windowMs
}));
```

### 11. **Backup Strategy**

#### Database Backup:
```bash
# Manual backup
pg_dump $DATABASE_URL > backup.sql

# Automated backup (script)
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
pg_dump $DATABASE_URL > "backup_$DATE.sql"
```
